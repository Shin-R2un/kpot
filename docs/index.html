ノートヴォルト

パスフレーズ一本で守る、1 ファイル完結の秘密管理ツール。 API キー・サーバー情報・パスワードを、ターミナルから安全に管理。

特徴

シンプルで強固な秘密管理

外部サービス不要。1 ファイルに全てを暗号化して保存。

🔒

最高水準の暗号化

Argon2id KDF（64 MiB / 3 回）+ XChaCha20-Poly1305 AEAD。AAD によるヘッダー完全性保護で改ざんを即検出。

🌱

リカバリシード

BIP-39 12/24 語シードフレーズまたは Crockford Base32 秘密鍵で、パスフレーズを忘れても復元可能（v0.3+）。

📦

バンドル転送

選択したノートを .kpb ファイルに暗号化してエクスポート。別パスフレーズで保護し、安全に受け渡し。

💻

REPL + ワンショット

対話型 REPL とシングルコマンド実行の両モード。TAB 補完・コマンド履歴・アイドルロックを搭載。

🔑

OS キーチェーン対応

macOS Keychain・GNOME Secret Service・Windows Credential Manager に DEK をキャッシュ。再起動後もパスフレーズ不要。

⚡

ゼロ外部依存 (UI)

ネットワーク不要。外部サービス不要。.kpot ファイル 1 つで完結。バックアップも cp 一発。

📋

クリップボード自動クリア

copy コマンドで秘密をクリップボードへ。設定可能な TTL（デフォルト 30 秒）後に自動消去。

💾

アトミック書き込み

.tmp → fsync → rename → .bak → fsync(dir) の 2 フェーズで書き込み。クラッシュ時も 1 世代前の .bak が残る。

🖥️

クロスプラットフォーム

Linux / macOS / Windows に対応。CI で ubuntu / macos / windows + クロスビルドを毎コミット検証。

暗号設計

多層防御の暗号アーキテクチャ

DEK（Data Encryption Key）分離設計で、パスフレーズ変更後もリカバリが継続して有効。

🔐 ヴォルト v2 (v0.3+) 暗号フロー

Argon2id

パスフレーズ + ランダム salt → KEK (32 B)。64 MiB / 3 回 / 並列 1 で GPU ブルートフォース耐性。

Wrap/Unwrap

KEK で DEK を XChaCha20-Poly1305 ラップ。WrapAAD でヘッダーフィールドをバインド。

XChaCha20-Poly1305

DEK でペイロードを暗号化。24 B nonce のランダム生成でノンス衝突確率を無視できるレベルへ。

AAD

ヘッダー全体（フォーマット・バージョン・両 Wrap・Cipher）をバインドし、任意フィールドの改ざんを検出。

Recovery

BIP-39 シード → PBKDF2-HMAC-SHA512 / 秘密鍵 → HKDF-SHA256 で別系統 KEK を生成。パスフレーズ変更不要で独立して有効。

📦 Bundle (.kpb) 暗号フロー

Argon2id

バンドルパスフレーズ + ランダム salt → KEK。ヴォルト KEK とは独立した鍵系統。

NewDEK

バンドル専用の乱数 BEK（Bundle Encryption Key）を生成。

Wrap

KEK で BEK をラップ。AAD にヘッダーフィールドをバインドし差し替え攻撃を防止。

XChaCha20-Poly1305

BEK でノートペイロードを暗号化。ヴォルトと完全分離した鍵で保護。

KPOT_BUNDLE_PASSPHRASE

バンドル用 env は KPOT_PASSPHRASE と別変数。ヴォルト開錠パスフレーズがバンドルに混入しない設計。

🛡️ セキュリティ対策一覧

Zero()

DEK・KEK・平文 JSON をスコープ脱出時に defer crypto.Zero() でゼロ埋め（ベストエフォート）。

/dev/shm

エディタ用一時ファイルを tmpfs 上に生成し、ディスク書き込みを回避。Close 時にゼロ埋め削除。

0o600

ヴォルトファイル・バンドルファイル・エクスポートファイルは全て所有者のみ読み書き可能。

Idle Lock

設定時間（デフォルト 10 分）無操作後に DEK をゼロ埋めしてプロセスを終了。

ConstantTimeCompare

パスフレーズ確認に crypto/subtle.ConstantTimeCompare を使用しタイミング攻撃を防止。

📁 ファイルフォーマット (.kpot)

JSON envelope

非暗号化ヘッダー（format / version / wraps / cipher）＋ Base64 暗号化ペイロード の 1 ファイル構成。

v1 / v2

v1: 直接 Argon2id 鍵でペイロード暗号化（後方互換）。v2: DEK ラップ方式、リカバリ wrap を追加。

Atomic write

.tmp → fsync → rename(vault→.bak) → rename(.tmp→vault) → fsync(dir) でクラッシュ耐性。

Note names

[a-z0-9._/-]、1〜128 文字、スラッシュでの階層化対応（例: ai/openai）。

アーキテクチャ

クリーンなパッケージ設計

循環依存なし。各パッケージが単一責任。テストは全パッケージで独立して実行可能。

cmd/kpot/main.go └─→ internal/repl (REPL セッション・全コマンドディスパッチ) ├─→ internal/vault (Open / Save / Create / Rekey / v1+v2) │ └─→ internal/crypto (Argon2id / XChaCha20-Poly1305 / Wrap) ├─→ internal/store (DecryptedVault CRUD / NormalizeName) ├─→ internal/bundle (.kpb 生成・解析・BEK ラップ) ├─→ internal/editor (tmpfs 上一時ファイル / エディタ起動) ├─→ internal/clipboard (Backend インターフェース / 自動クリア Manager) ├─→ internal/notefmt (front-matter Render/Strip / プレースホルダー) └─→ internal/tty (パスフレーズ読み取り / リカバリ表示 / SharedStdin) └─→ internal/keychain (macOS / Linux / Windows 別バックエンド) └─→ internal/recovery (BIP-39 / Crockford Base32 / PBKDF2 / HKDF) └─→ internal/config (~/.config/kpot/config.toml TOML パーサー)

インストール

すぐに使い始められる

ワンライナーインストーラー、go install、またはソースビルドで。

# ワンライナーインストーラー（最新リリースを /usr/local/bin へ）
curl -fsSL https://raw.githubusercontent.com/Shin-R2un/kpot/main/install.sh | sh

# インストール確認
kpot version
# PowerShell（管理者権限不要）
irm https://raw.githubusercontent.com/Shin-R2un/kpot/main/install.ps1 | iex

# または Scoop（コミュニティバケット）
scoop install kpot
# Go 1.18 以上が必要
go install github.com/Shin-R2un/kpot/cmd/kpot@latest

# インストール確認
kpot version
# リポジトリをクローンしてビルド
git clone https://github.com/Shin-R2un/kpot.git
cd kpot
make build          # → ./kpot バイナリ
make test           # 全テスト実行
make install        # GOPATH/bin へインストール

# クロスコンパイル例
GOOS=windows GOARCH=amd64 go build -o kpot.exe ./cmd/kpot
GOOS=darwin  GOARCH=arm64 go build -o kpot-mac  ./cmd/kpot

クイックスタート

# 1. ヴォルトを初期化（リカバリシードが表示される。必ず書き留める！）
kpot init personal.kpot

# 2. REPL を起動して最初のノートを作成
kpot personal.kpot
kpot:personal> note ai/openai
kpot:personal> ls
kpot:personal> read ai/openai
kpot:personal> exit

# 3. ワンショットで読み取り
kpot personal.kpot read ai/openai

# 4. 設定ファイル（任意）
mkdir -p ~/.config/kpot
cat > ~/.config/kpot/config.toml <<EOF
editor                = "nvim"
clipboard_clear_seconds = 45
idle_lock_minutes     = 15
keychain              = "auto"
EOF

✅ DEK 分離アーキテクチャ

v2 の passphrase_wrap / recovery_wrap 独立構造。RekeyV2 が DEK を保持しリカバリを継続有効に維持。

✅ 完全な改ざん検出

AAD に全ヘッダーフィールドをバインド。memory_kib 改ざん・nonce 改ざん・payload 改ざん全てを AEAD で検出。

✅ BIP-39 + Crockford Base32

2 形式のリカバリキー。NormalizeMnemonic でコピペノイズ吸収。O→0 / I→1 の自動修正実装。

✅ mergeNotes 128 文字制限保護

truncatedConflictBase ヘルパーで suffix 保持・prefix 短縮。長名ノートのコンフリクト後も Get で到達可能。

✅ KPOT_BUNDLE_PASSPHRASE 分離

バンドル用 env を KPOT_PASSPHRASE から完全分離。ヴォルト開錠パスフレーズがバンドルに混入しない。

✅ Windows /dev/tty 修正

recovery_unix.go / recovery_windows.go にビルドタグ分割。Windows は IsStdoutTTY 確認済みの os.Stdout を使用。

⚠ 既知軽微バグ: envWarnOnce 共有

ReadPassphrase と ReadBundlePassphrase が同一 sync.Once を共有。同一プロセスで両 env を使うと片方の警告が表示されない。後続 PR で修正予定。

⚠ crypto.Zero はベストエフォート

Go の GC がオブジェクトを移動する可能性があり、ゼロ埋めは保証ではなくベストエフォート。Go のランタイム制約による。
